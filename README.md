## Reflection

### What was the problem you were solving in the projects for this course?
The primary problem addressed in this course was determining the most efficient way to store, manage, and retrieve course data. In Project One, I evaluated multiple data structures by comparing their run-time performance and memory usage. In Project Two, I developed a program that reads, sorts, and prints course information in alphanumeric order, ensuring both accuracy and efficiency in data handling.

### How did you approach the problem? Consider why data structures are important to understand.
I approached the problem by first analyzing the requirements and then evaluating which data structures would best meet those needs. I compared vectors, hash tables, and binary search trees to understand their strengths and weaknesses. Data structures are essential because they directly impact a program’s performance, scalability, and efficiency. Selecting the appropriate structure allowed me to optimize sorting and searching operations while maintaining logical organization within the code.

### How did you overcome any roadblocks you encountered while going through the activities or project?
One of the main challenges was implementing sorting and ensuring that course data was structured correctly. I addressed these challenges by breaking the problem into smaller components and testing each part individually. I also used debugging techniques and revisited course materials to reinforce my understanding. This step-by-step approach made complex problems more manageable and helped ensure accuracy in my final program.

### How has your work on this project expanded your approach to designing software and developing programs?
This project reinforced the importance of planning and design before coding. I now place greater emphasis on evaluating different approaches and selecting appropriate data structures early in the development process. It also improved my ability to think critically about efficiency and performance, which are key factors in building effective software solutions.

### How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?
Through this project, I developed better coding practices focused on clarity and maintainability. I now prioritize meaningful variable names, consistent formatting, and modular design. Writing code in smaller, well-defined functions has made my programs easier to read, debug, and modify. These improvements ensure that my code is more adaptable for future enhancements or changes.
